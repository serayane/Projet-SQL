# Documentation SQL

## Pourquoi des requêtes indépendantes ?
Les requêtes ont été conçues pour fonctionner sur une base relationnelle
ayant une structure logique similaire, même si les noms exacts des champs
ou tables diffèrent légèrement.

## Vues
Les vues permettent de simplifier les requêtes complexes et d'améliorer
la lisibilité du code SQL.

## Fonctions
Les fonctions SQL facilitent la réutilisation de traitements complexes
comme les calculs de montants.

## Triggers
Les triggers assurent le respect des règles métiers directement au niveau
de la base de données.
